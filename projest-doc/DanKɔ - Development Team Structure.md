## 3. Hiring Process

### 3.1 Where to Find Talent in Ghana

| Platform | Best For | Cost |
|----------|----------|------|
| LinkedIn | Senior roles | Free - GHS 500/post |
| Jobberman Ghana | All levels | GHS 200-800/post |
| DevCongress Slack | Developers | Free |
| GDG Ghana | Flutter devs | Free |
| Twitter/X Tech Ghana | All roles | Free |
| University Career Centers | Junior roles | Free |
| Referrals | All roles | Bonus-based |

### 3.2 Hiring Timeline

| Week | Activity |
|------|----------|
| Week 1 | Post job descriptions, source candidates |
| Week 2 | Review applications, initial screening calls |
| Week 3 | Technical assessments, portfolio reviews |
| Week 4 | Final interviews, reference checks |
| Week 5 | Offers, negotiation, onboarding prep |
| Week 6 | Team onboarding, project kickoff |

### 3.3 Interview Process

**Round 1: Screening (30 mins)**
- Background and experience review
- Motivation and availability
- Salary expectations

**Round 2: Technical Assessment**
- Flutter Dev: Build a small feature (take-home, 4 hours)
- Backend Dev: API design challenge (take-home, 4 hours)
- Designer: Design a screen based on brief (take-home, 4 hours)
- QA: Test case writing exercise

**Round 3: Technical Interview (60 mins)**
- Deep dive into assessment
- Problem-solving scenarios
- Architecture discussions
- Code/design review

**Round 4: Culture Fit (30 mins)**
- Team collaboration style
- Communication approach
- Conflict resolution
- Long-term goals

### 4.3 Development Workflow

```
Feature Request
      ↓
Design (if UI) → Design Review → Approved
      ↓
Create Branch → Development → Self-Test
      ↓
Pull Request → Code Review → Approved
      ↓
Merge to Develop → QA Testing → Approved
      ↓
Merge to Main → Deploy to Staging → Stakeholder Review
      ↓
Deploy to Production
```

### 5.3 Contract Types

**Option A: Full Employment**
- Pros: Dedicated team, IP protection, control
- Cons: Higher cost, benefits overhead
- Best for: Core developers

**Option B: Contract/Freelance**
- Pros: Flexibility, lower commitment
- Cons: Less control, availability risks
- Best for: Designer, DevOps, QA

**Option C: Agency Partnership**
- Pros: Turnkey solution, managed team
- Cons: Higher cost, less direct control
- Best for: If lacking technical oversight

## 8. Risk Mitigation

### Team Risks & Solutions

| Risk | Probability | Mitigation |
|------|-------------|------------|
| Key person leaves | Medium | Documentation, knowledge sharing |
| Skill gaps discovered | Medium | Training budget, pair programming |
| Communication issues | Low | Regular 1-on-1s, clear channels |
| Scope creep | High | Strict change control, PM oversight |
| Burnout | Medium | Sustainable pace, time off policy |
